# 云屿摄影小程序 AI Agent 免费优先升级统一方案

> 整合自外部方案目录 11 份分析文档，结合当前代码 v1.72 实际状态，产出可落地执行方案。

## 一、目标边界

### 1.1 升级范围
- **核心目标**: 在不引入必须付费的新模型服务前提下，尽可能提升 AI Agent 能力
- **免费优先**: 默认保留/兼容当前免费模型 API (Agnes APIHub / gpt-4o-mini)
- **降级策略**: 所有新增能力均可降级到纯本地逻辑或关闭

### 1.2 不在范围内
- 不引入付费向量数据库 (MongoDB Atlas M10+)
- 不引入需要额外服务器的 WebSocket 流式方案
- 不引入付费 embedding API 作为必须依赖
- 不推送、不部署、不公开发布

## 二、MVP 升级清单 (免费优先路线)

### Phase 1: 系统提示词优化 (零成本)
1. **动态提示词分层**: Layer 0 核心 (150 tokens) + Layer 1 业务场景 (300-500 tokens) + Layer 2 按需数据 (0-800 tokens)
2. **员工/类型列表压缩**: JSON 数组 (~1600 tokens) → 紧凑文本 (~50 tokens)，节省 60%
3. **按查询类型裁剪**: 闲聊不发工具指令，查询不发写入指令

### Phase 2: 智能模型路由 (降成本 50-80%)
1. **任务复杂度分类**: trivial (问候) → 简单模型, simple (查询) → 中等模型, complex (截图识别) → 强模型
2. **动态 maxTokens**: 150/400/1200 按任务级别
3. **模型降级链**: 主模型 → gpt-4o-mini → gpt-3.5-turbo (429/5xx 自动降级)

### Phase 3: 打字机效果 (零成本)
1. **前端模拟流式**: 自适应速度 (30-50ms/字)，标点感知停顿
2. **跳过按钮**: 长文本可点击跳过动画
3. **感知等待时间降低 60-80%**

### Phase 4: 会话记忆增强 (低成本)
1. **本地滚动摘要**: 消息超 20 条时压缩旧消息为摘要
2. **关键词记忆提取**: 从对话中提取客户名、日期、类型等关键信息
3. **记忆注入提示词**: 将历史摘要注入系统提示词

### Phase 5: 知识库关键词检索 (零成本)
1. **本地知识库结构化**: 拍摄SOP、销售话术、品牌资料、财务规则
2. **关键词匹配检索**: 用户消息匹配知识条目，注入提示词
3. **分类触发**: 按话题类别自动加载相关知识

### Phase 6: 技能配置系统 (低成本)
1. **技能定义**: 纯配置 (触发关键词 + 系统提示 + 工具白名单)
2. **技能匹配**: 关键词匹配 + 优先级排序
3. **内置技能**: 档期查询、截图录单、知识检索、客户跟进

## 三、风险与回滚

### 3.1 风险
- 动态提示词可能导致 AI 回复质量波动
- 模型路由分类错误可能用错模型
- 打字机效果在低端设备可能卡顿

### 3.2 回滚方案
- 每个 Phase 独立 feature flag，可单独关闭
- 保留原有 DEFAULT_CONFIG 作为 fallback
- 代码改动通过 git commit 可精确回滚

## 四、验收标准
1. 现有 8 个 Agent 动作全部正常工作
2. 新增能力不影响现有 API 调用
3. 免费额度使用量不增加 (云函数、DB 读写)
4. 所有改动有本地 git commit 记录
5. 1000+ bug-hunt 巡检完成
