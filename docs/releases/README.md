# Release Notes Archive

This folder preserves version-by-version release notes for $full.

Principles:

- Keep historical release copy append-only by default.
- Do not overwrite earlier README or release text unless fixing mojibake, leaked secrets, broken links, clearly wrong commands, or security risks.
- Put detailed version notes in docs/releases/vX.Y.Z.md and GitHub Releases.
- Keep README.md focused on the current entry point.

Public releases page: https://github.com/lychee20000105/photography-studio-schedule-miniprogram/releases
