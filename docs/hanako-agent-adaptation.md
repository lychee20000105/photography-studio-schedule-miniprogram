# 云屿小猫 Agent 参考 Hanako 的迁移方案

记录日期：2026-06-11

## 参考来源

- GitHub：https://github.com/liliMozi/openhanako
- 仓库描述：Personal AI agent with memory, personality, and autonomy
- 许可证：Apache-2.0
- 当前可见包版本：`hanako@0.309.2`
- 说明：本次只参考其架构思路，不直接复制底层代码；后续若迁移具体实现，需要保留许可证与来源说明。

## 可迁移能力

1. 人格层
   - Hanako 通过人格模板塑造不同 Agent。
   - 云屿小猫已接入 `ops_cat/gentle_cat/strict_cat/sales_cat`，下一步可把人格独立成可导入导出的 Agent 配置。

2. 工具层
   - Hanako 把文件、浏览器、搜索、命令等能力拆成工具。
   - 云屿小猫应继续采用受控动作 JSON：`query_schedule/create_order/create_orders/join_order/create_item/create_rest/add_note`。
   - 后续新增联网搜索时必须走云函数代理，并记录来源、时间和权限。

3. 技能层
   - Hanako 支持 Skills 和技能包。
   - 云屿小猫可先做“摄影工作室技能包”：截图录单、明日安排、客户跟进、店内知识检索、员工规则解析。

4. 记忆层
   - Hanako 强调本地优先和多 Agent 独立记忆。
   - 小程序端适合分三层：本机会话记忆、员工可见业务记忆、店内知识库。三层必须权限隔离。

5. 安全层
   - Hanako 有沙盒和权限模型。
   - 云屿小猫必须继续保持禁止直接删除、作废、收款、退款、发工资、改提成、审核通过等高风险动作。

## 当前已落地

1. 小猫 Agent 有版本号与内置更新日志。
2. 多会话保存在本地。
3. 订单、事项、休息、小记使用受控动作。
4. 新增 `join_order` 安全动作，只允许当前员工把自己加入订单参与人。
5. 访客模式完全隔离真实数据，访客新增只进本机临时缓存。

## v1.86 新增落地

1. 技能注册表：新增 `work_ai_agent_registry.js`，把档期查询、订单录入、图片录单、改期纠错、财务、工资审核、小记事项、休息请假、客户跟进和知识问答拆成内置技能。
2. 工具白名单：小猫每轮对话按技能生成允许动作，模型返回不属于本轮技能的动作不会执行。
3. 轻量记忆：新增 `work_ai_agent_memory.js`，只做当前会话和页面上下文压缩，不自动写长期记忆库。
4. 审计底座：新增 `work_agent_audit_model.js`，AI 写入动作除团队小记外，额外尝试写 Agent 审计流水。
5. 配置自由度：管理端 AI 配置支持文本模型与视觉模型分开配置，并拆出 Mimo/自定义预设。

注意：桌面 Hanako 的文件系统、终端、浏览器控制能力不迁移到微信小程序；小程序只迁移 Agent 分层和受控业务工具模式。

## 后续待落地

1. 流式回复
   - 小程序端建议先做“分段渲染/打字机效果”，服务端保留非流式。
   - 真流式需要云函数或自建 HTTPS 服务支持 SSE/WebSocket，再由小程序分片消费。

2. 联网搜索
   - 新增 `web_search` 工具动作。
   - 仅允许管理员配置搜索提供方。
   - 搜索结果必须展示来源链接和检索时间。

3. Skill 配置
   - 建立 `agentSkills` 配置表。
   - 每个 skill 包含：名称、触发词、系统提示、允许动作、权限范围、版本号、更新日志。

4. 店内知识检索
   - 上传资料后生成结构化条目。
   - 每条知识有可读角色/员工/管理员范围。
   - 小猫回答时只检索当前员工可见资料。

5. Agent 可抽离
   - 把人格、工具动作、权限、版本、技能、知识检索分成独立配置。
   - 保留小程序适配层，方便后续迁移到其他系统。
