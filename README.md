# 摄影工作室档期小程序

以云屿摄影小程序为案例的微信云开发开源项目

当前稳定案例版本：`v1.1.0`

案例发布时间：`2026-05-25`

v1.2.0 规划状态：`架构预告 / 实施规划`

这是一个面向摄影工作室的档期管理小程序开源项目，用于团队内部档期查看、客户预约记录、订单进度协同、员工提成和拍摄服务展示。当前仓库以“云屿摄影”的实际业务小程序作为案例，适合小型摄影团队二次开发为自己的生日跟拍、百日宴、婚礼跟拍、订婚宴、寿宴、乔迁跟拍、写真、外景约拍、活动商拍、艺术肖像等档期系统。

## v1.2.0 更新预告

v1.2.0 计划把现有“档期 + 订单 + 员工提成 + 工资结算”能力，升级为更适合摄影工作室真实经营的业绩与财务协同模块。

核心方向：

- 新增“业绩”底部导航入口。
- 新增 `bx_work_payment` 收款账本，按实际收款月份统计业绩。
- 新增 `bx_work_commission` 提成账本，支持 70/30 先发与冻结释放。
- 工资页改为从提成事件汇总，展示先发、释放、扣回和待释放。
- 管理员从小程序“我的”页免密码进入管理中心。
- 管理接口基于 `openid + STAFF_IS_ADMIN` 后端二次校验。

## 案例说明

本项目不是只服务某一个品牌的封闭系统，而是一个可复用的小程序案例工程。“云屿摄影”相关名称、业务类目、AppID 和云环境 ID 均来自案例配置，二次使用时可以替换为自己的摄影工作室品牌、服务项目和微信云开发环境。

## 核心功能

- 内部档期：支持月历彩色标签、全店/我的筛选、订单档期、事项档期、休息日期。
- 订单记录：可记录客户、电话、地点、来源、订单金额、定金、尾款、后期加选/产品金额、参与人和附件截图。
- 员工提成：员工按手机号绑定，可按员工+岗位设置提成规则；订单参与人可选择按比例、固定金额、手动金额或不计提成。
- 工资结算：订单完成后提交管理员审核，通过后进入待结算；管理员可按员工单独确认已发工资，已结算数据不重复计算。
- 团队查看：非参与员工只看脱敏档期，参与员工可看订单金额但不可看其他员工提成。
- 样片展示：展示跟拍纪实、艺术肖像、写真作品、外景约拍、活动商拍等作品分类。
- 拍摄服务：维护服务价格、亮点、服务内容、行程安排和交付说明。
- 后台管理：支持预约名单、核销、导出、内容、样片、服务、管理员、员工、订单类型、审核和工资结算维护。
- 企业微信群提醒：支持通过云函数每天推送当日档期摘要到企业微信群。
- 企业微信配置：上线后可在后台“企业微信机器人”页面填写 Webhook，并测试发送。

## 后台账号

后台超级管理员初始账号：`admin`

首次初始化不再内置默认密码。部署前请在云函数环境变量中配置 `B00_ADMIN_INIT_PASSWORD`（或 `ADMIN_INIT_PASSWORD`），要求 12-30 位并至少包含大小写字母、数字、符号中的 3 类。

如未配置环境变量，系统会生成不可预测临时密码并写入数据库，但不会在日志中输出明文；请通过安全渠道重置后再交付使用，并按员工职责创建普通管理员。

## 快速开始

1. 使用微信开发者工具导入项目根目录。
2. 确认 `project.config.json` 中的 AppID。
3. 确认 `cloudfunctions/mcloud/config/config.js` 中的 `CLOUD_ID`。
4. 在云开发控制台创建或绑定对应云环境。
5. 上传并部署 `mcloud` 云函数。
6. 上传并部署 `autoSendDaily` 云函数。
7. 初始化后台数据后登录后台配置员工、订单类型、档期、样片、拍摄服务和企业微信机器人。
